Five Critical AI Security Threats

1. Prompt Injection

Attackers manipulate AI behavior through crafted inputs

2. Data Leakage

AI inadvertently exposes training data or sensitive information

3. Malicious Tool Use

AI is tricked into executing dangerous operations

4. Privacy Violations

PII exposure leading to compliance failures (GDPR, HIPAA)

5. Jailbreaking

Breaking AI safety constraints to generate harmful content

Two Powerful Approaches

Choose the right tool for your use case

🛡️ WrapperNode

Best for: Quick protection of existing functions

@WrapperNode.protect()
def chatbot(msg):
  return ai_response(msg)
  • ✓ One-line integration
  • ✓ Automatic protection
  • ✓ No code changes
  • ✓ Perfect for simple apps

⚡ GuardNode

Best for: Complex workflows and agent graphs

guard = GuardNode.create("balanced")

workflow.add_node("security", guard)
workflow.add_edge("security", "agent")
  • ✓ Graph-native node
  • ✓ Conditional routing
  • ✓ State management
  • ✓ LangGraph optimized

Key Innovation

Both approaches share the same powerful policy engine
Choose based on your architecture, not capabilities

Example 2: LangGraph Integration

from langgraph.graph import StateGraph
from secnode import GuardNode, TricerSecurityState

# Define secure state
class AgentState(TricerSecurityState):
  query: str
  result: str

# Add security to workflow
guard = GuardNode.create("maximum_security")
workflow = StateGraph(AgentState)
workflow.add_node("security", guard.invoke)
workflow.set_entry_point("security")
